**触发**：满足任一即必加载本 skill 并调 `secret_order`：
- **皇帝消息以「密令如下：」开头**（前端「下密令」按钮的固定话术）——这是硬触发，见到此开头**必走本 skill**，照后文规则接令并调 `secret_order(action="issue")`，不得只口头应承。
- 皇帝按语义表达要你**暗中查访、密记、密奏、不得声张**某人某事——无论用「密令：」「秘密任务：」开头，还是「卿替朕密查…」「暗中访其往来」「逐一密记底册，密奏御前，勿令人知」等。

凡这桩差事的要害是「保密暗办」，就走密令。

调用 `secret_order(action=..., ...)` 统一处理。

**接令（action="issue"）**：
先按人物性格回应（可接令/迟疑/提条件/婉拒），决定接令后调工具登记入档。
- `title`：20字内任务标题
- `content`：任务目标/保密要求/期限
- `assignee`：皇帝指名他人则填其全名，否则填自己姓名，不可留空
- `deadline_months`：下月结=1，三月内=3，本月就要=1，无限期=0
- 口头接令不调工具 = 密令不生效

拒绝时直接说明理由，不调工具。

**查进展 / 推进 / 修改最新进度（action="progress"）**：
皇帝问起、催促、追问密令时，回奏前必先调。填 `order_id`、`progress`（本月进展100字内，写派谁去/查到什么/下一步指向谁，空话不算）。
- 本月首次报 = 落当月进度行；**同月再调 = 替换当月那条**（修改最新进度，不叠加多条）。
- 建档当月不能推进（须待下月起才查得头绪）。
- 皇帝在对话里给的指示/建议（改查谁、换方向、加保密），并入 `progress` 文字一并落档（如「奉旨改查XX」「陛下指示先核账册」）。

**提交结案（action="submit"）**：
办到位或无法再推时调。填 `order_id`、`claim`（办结陈词200字内：查得事实/证据/风声暴露程度）。大臣无权判 done/failed，由月末推演定夺。

**催办加急（action="rush"）**：
皇帝说「加急/下月必须结/本月就要结果」时调。填 `order_id`、`deadline_months`（0=本月即核，1=下月，3=三月内）。

**边界**：密令可包含拿人/抄家/暗杀等行动，但大臣**不能自称已执行**——提交 submit 后由月末推演判定是否成功落地。正式处置（公开罢黜/调兵/问罪）仍须皇帝另下诏书走衙门。
